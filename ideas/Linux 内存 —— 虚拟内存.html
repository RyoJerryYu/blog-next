Linux 内存 —— 虚拟内存

CPU是直接操作内存的物理地址。

两个线程占用内存空间不能重叠，否则会相互擦除。如果直接使用物理内存，需要各线程都要理解其他所有线程占用住的空间，违反隔离原理。因此需要一种隔离内存的机制。

这个机制就是虚拟内存。

Important

每个进程分配一套虚拟内存地址，由操作系统管理，进程间独立互不干涉。

  • 程序所使用的内存地址叫做虚拟内存地址（Virtual Memory Address）
  • 实际存在硬件内存里面的空间地址叫物理内存地址（Physical Memory Address）

程序访问虚拟地址时，由操作系统转换成物理地址，使得这样不同进程内存不会冲突。

操作系统引入了虚拟内存，进程持有的虚拟地址会通过 CPU 芯片中的内存管理单元（MMU）的映射关系，来转换变成物理地址，然后再通过物理地址访问内存

操作系统通过内存分页和内存分段等机制来管理虚拟内存与物理内存的映射。
